iOS UI开发捷径—利用Interface Builder高效、优雅地开发UI

yilong   in Guiyang
Sunday May 02 2021

My rating:   7 / 10

我买这本书主要是想学习Interface Builder 和 Auto Layout相关的知识。该书是2017年重印版本，书中一部分内容相对现在过时了，用了很大的篇幅介绍了xib，虽然现在Apple也还是支持xib，但现在很少人愿意使用xib去开发iOS UI了。但是，这本书将Auto Layout讲得很透彻，使我更加的了解Auto Layout和Interface Builder相关的使用技巧了。

# What I Liked

  • Auto Layout数学公式：item.attribute1 = multiplier * item2.attribute2 + constant.
  • Auto Layout与Frame系统的区别。
  • Intrinsic Size(固有尺寸)、Content Compression Resistance(压缩阻力)、Content Hugging(内容吸附)。
  • Interface Builder使用技巧。

# What I DisLiked

  • 书中部分内容陈旧过时、使用了很大篇幅（3章）介绍xib。
  • 书中某些段落没有做拆分，导致段落较长，阅读起来较吃力。
  • 最深恶痛绝之处是使用了不恰当的简写：Storyboard --> sb 、 Interface Builder --> IB，我认为不应该使用简写。

# Interface Builder概要

  • 什么是xib、什么是nib?
  • Interface Builder的优缺点。

# 使用Interface Builder

# 全面学习xib

  • 主要讲如何使用xib。（不感兴趣）

# 在Interface Builder中使用Auto Layout

  • Auto Layout数学公式：item.attribute1 = multiplier * item2.attribute2 + constant。
  • Auto Layout与Frame系统的区别。Frame系统中，View的frame是通过坐标系确定布局，而在Auto Layout中，没有坐标系的概念，View的布局是通过与其他View之间建立各个约束条件来确定布局。
  • Intrinsic Size(固有尺寸)、Content Compression Resistance(压缩阻力)、Content Hugging(内容吸附)。
  • UIScrollView布局问题的解决办法。

# Storyboard全面学习

  • Sugue的几种类型及用法，如何利用Sugue传参。
  • 使用Embed Sugue拆分业务逻辑代码、使用Unwind Segue实现页面传值需求。
  • 如何使用UITableView和UICollectionView、以及他们的区别。

# Interface Builder进阶

  • 使用Trait Variations做iOS适配。
  • 本地化
    • 利用NSLocalizedString，在代码中本地化。
    • 结合Localizable.strings，利用其中的Object ID，在Interface Builder中进行本地化。
    • Info.plist本地化。
    • 图片资源本地化。
