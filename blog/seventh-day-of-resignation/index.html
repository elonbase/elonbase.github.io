# 面试问题

# 笔试题
  • 1.请简述TCP三次握手过程，TCP和UDP的区别，并讲诉他们分别有什么应用场景。
  • 2.请介绍一下C++编译、链接的过程以及中间产物。
  • 3.有一个t_student表，包含两个字段name, age,请写出查询表中年龄大于16岁的学生sql,并按年龄从大到小排序。
  • 4.大家都知道范型编程，而C++的模板和STL很好支持了范型编程，请举例说明模板和STL是如何支持范型编程的。
  • 5.现在大家都在谈面向对象的开发，面向对象的基本特征是封装、继承、多态，请结合你曾经开发的项目谈谈对这三个特征的理解。
  • 6.请结合您曾经参与的项目谈谈你对分层和模块化以及抽象的理解。
  • 7.请使用冒泡排序编写一个排序算法（Java/C/C++）
  • 8.我们知道多线程可以实现异步程序，请问异步程序存在哪些优点和缺点。
  • 9.下面这个函数当传入5的时候返回值是多少。(写了1，正确答案：8)
int foo(int n) {
  if(n < 2) 
    return n
  return foo(n-2) + foo(n-1)
}
  • 10.请用C/C++实现两个函数，分别可以获取int数据的高8位和低8位的值。

你没看错，这就是前端笔试题，我做了1、3、5、8、9题。我很怀疑这个公司的水准。

# 面试官A（感觉是个新手）
  • 如何快速的搭建一个项目？
  • 你用过一些跨端的技术吗？
  • 你有用过CSS的一些动画吗？
  • 你在前端项目中有没有用过什么牛逼的技术？（我觉得前端的技术不是很牛逼，就说了富文本编辑的二次开发）
# 面试官B（技术总监）
  • 为什么从上家公司离职？
  • 你用过一些跨端的技术吗？
  • React跟Vue的区别是什么？
  • 你了解或开发过小游戏吗？（没有）
  • 你的期望资新是多少？

不止这些问题，只记得这几个了。最近都神经紧绷，睡眠质量不好，记忆力貌似也下降，在回答面试官的问题时，我连我上上份工作的时间都忘了。这一点让他对我说的话的真实性产生了一些怀疑。

# 我的问题

  • 贵公司有多少人，你负责的团队有多少人？（他不知道公司有多少人，他管理了近10人的公司）
  • 贵公司有可以看到外面风景的工位吗？（他：我也想有这样的工位）

# 面试总结

以后再也不投这家公司了。

从做题到面试完成，大概1个小时左右，中间等的时间估计有10多分钟。

HR： 长的漂亮，但还是没表情。（我到底要怎样？）

面试官A：人很谦虚，是技术总监派来试探的小兵。只问做过没有，不往深了问。

面试官B（技术总监）：年纪估计和我差不多，脖子上又一块淤青，他最近应该挺快活的。从谈话当中感觉我不是他们想要的。到最后时问我的期望薪资，我还是没有说，我让他们决定，而且我说得有一点拽：

我相信贵公司有自己的薪酬体系和人才评判的标准，你可以给一个Boos上薪资区间的一个数，如果我觉得OK，那么我就来，如果不OK，那么希望下次有机会再合作。

最后让我回去等消息，我知道这次又跪了。但是，我一点也不遗憾。

其实，今天让我焦虑的，是昨天的面试结果还没有通知，我很担心挂掉了，因为它是我目前面试的这几家中最满意的，而且，到现在为止，我还没收到一家的Offer。

我开始在怀疑我的能力，是不是我盲目乐观，没有看到面试官看到的一面，我自我感觉良好，其实给面试官的印象却正好相反。我现在也拿捏不准面试官的评判标准和反馈的真假。我只能凭感觉去猜测了。

# 面试预告

  • 云上贵州。今天接到云山贵州HR的电话，通知6号下午面试。这家公司是离职前几天投的，后面没消息，打电话过去问，他们还在收集简历，要到月初才开始，果然。说明应聘人数较多，可能会碰到熟人。大型面试现场，这种体验是没有过的。好好准备吧。
  • 中国电信。8号早上。前几天Boos上投的。之所以有这个面试机会，我想是我们在Boos上的问答，我的回答打动了他吧。

Boss：考虑换工作的原因是什么？

我：想到更具有挑战性的环境中工作，获得职业能力上的提升。

Boss：主要使用的技术栈是什么？最值得自豪的成绩或成长是什么？

我憋了很久,答：主要使用Vue,认为最大的成长是，意识到前端开发不仅仅只是一个工程岗位，做为与用户交互的第一场景，UI设计、用户体验、无障碍设计能实现使世界变美好。

Boss给了一个大大的赞的表情。
