# Summary

# 爽约成都之行

从去年国庆节结婚到现在，和老婆没有蜜月旅行。前几个月打算好的5·1节去成都旅行，我在临行的前一天晚上洗澡时，突然决定不去了。 我想利用这几天时间推进我的Side Project进度。当然，这遭到老婆的不满与反对，但是我还是决定了。

她知道我的脾气，我决定的事情很难说动我，而且强求也没意思。 于是她和她的妹妹去了，我知道我这样临时变卦很不好，也很自私。 但是我为了完成自己的心愿，我愿意舍弃这次旅行，甚至惹她不开心。 我希望我能如愿以偿，对她的亏欠，以后再规划一次旅行来弥补吧。

# 5·1假期生活

  • 5月1日 ～ 5月2日看完了《iOS UI开发捷径》一书。这本书貌似是在离职前买的，因为找工作的事情，一直耽搁到现在才有时间看，Side Project也是最近找到工作后才继续的。

  • 5月3日 ～ 5月4日看完了《精益创业》一书。看完基本技术书籍后，总要看一本非技术类的书籍。就像吃饭不能光吃肉，也要吃点蔬菜，这样的荤素搭配感觉才营养健康。

  • 去无印良品买了2件同款不同色的T恤衫，以后尽量减少在穿衣上做决策的时间。虽然我没有在穿衣的决策上纠结，但是为了保持个人形象风格统一，同类型的衣服一般我会买多件。

  • 5·1期间，我大多时间都是待在家里，没有做饭，靠泡面、鸡蛋和酸奶裹腹。我甚至有些享受一个人的生活，不受打扰的思考做事。晚上困了，可以不刷牙就能直接去睡觉。

# 关于新工作

假期回来，上线了节前的2个需求，又开始了同类型的2个需求。

我觉得这些所分配给我需求对我没有太大的挑战，而且都是Fork目前公司已有的项目做修改，只是项目所属的甲方不一样而已。

一次一个刚来不到2个月的同事一起吃饭聊到，他说：“我感觉公司就是一个外包公司，我才来不到2个月，陆陆续续就做了4个项目，这个数量是我在上一家公司一年所做项目的数量”。

他其实也没有说错，只要不是在开发自有产品或在自有产品上做迭代的开发，承建其他甲方的项目的开发，可以认为是外包。公司其实是有自己产品，但是公司并没有将其SaaS化，而是基于现有产品做定制开发及私有化部署。 这样的工作性质其实跟我们所说的外包很相似。

这种重复劳动和无挑战的工作，那么对于员工的能力成长是有所损害的。我到时也会根据自己的情况，再评估自己对该企业的存在价值。
