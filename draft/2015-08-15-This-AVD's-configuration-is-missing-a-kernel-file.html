这个问题其实困扰我很久了。今天终于鼓起勇气一定要战胜它。Emulator error: This AVD's configuration is missing a kernel file !!

于是，先百度一把，但是百度知道上的答案还是不能解决我的问题。最后在stackoverflow.com—http://stackoverflow.com/questions/9712605/emulator-error-this-avds-configuration-is-missing-a-kernel-file发现了一个哥们建议。

于是照着他说的，用了emulator @SonyTabletS -verbose，把SonyTabletS替换成了自己的模拟器名称4.1.2，回车。然而返回的emulator: found ANDROID_SDK_ROOT:的路径并不是我在环境变量中设置AANDROID_SDK_HOME的路径，两个路径既然不一样。然后我打开系统环境变量设置，发现在用户变量中发现了一个ANDROID_SDK_ROOT的变量名指向了了刚刚命令行返回的路径。哇靠，我想我应该找到原因了，于是把它删了。重启Eclipse，运行Android模拟器，啊哈，不报错了，完美的运行起来了。

此时的心情，用刚治好的便秘来形容最合适不过了:)。
